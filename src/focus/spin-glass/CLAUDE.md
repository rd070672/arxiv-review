## 実行手順
--------------------------------

### Step 1. main.pdf の精読
まず mainフォルダ内の`main.pdf` を最優先で精読し、以下を抽出すること。

1. この論文が扱う中心テーマ
2. 解決しようとしている科学的問い
3. 著者が主張する新規性
4. 用いている主要手法
   - 実験
   - 理論
   - 計算
   - データ解析
5. 最も重要な結果
6. その結果を支える証拠
7. 論文内で暗黙的または明示的に残されている限界・未解決点
8. 関連研究を探すためのキーワード候補

ここでは、特に次の形式で整理すること。
- 主張
- 根拠
- 比較対象
- 残る論点

### Step 2. references 内の sub 論文の確認
次に `references` フォルダ内の `sub-*.pdf` を読み、
各論文が main 論文に対してどの役割を果たすかを判定すること。

役割カテゴリは以下を用いること。
- 背景を与える論文
- main 論文の主張を支える論文
- 別視点から比較できる論文
- 異なる手法を使う論文
- 他材料系や他分野へ広げる論文
- 応用やデバイス接続を示す論文

重要:
- sub 論文を独立した主役として扱わない
- 「この sub 論文は main のどの論点を理解するのに必要か」を軸に評価する
- 関連が弱いものは、無理に使わなくてよい

### Step 3. main 論文から検索キーワードを設計
main.pdf の内容を踏まえて、arXiv 検索用のキーワードを 3〜8 個程度設計すること。

キーワードは単語列ではなく、検索意図を持つ形で設計すること。
たとえば以下を含めてよい。
- 中心物質名
- 中心現象名
- 主要手法
- main 論文で議論される機構名
- 競合概念や比較対象
- 応用先

検索キーワードは、
「main 論文の議論を深めるために必要な論文が見つかるか」
という観点で選ぶこと。

### Step 4. arXiv で追加論文を検索
以下を使って検索すること。
`https://arxiv.org/search/cond-mat`

必要に応じて cond-mat 配下の relevant category を広げてもよいが、
中心は cond-mat search とすること。

検索では 10~20 本程度の関連論文を選ぶこと。
ただし重要なのは件数ではなく relevance である。

選定基準:
- main 論文の主張理解に直接役立つ
- 背景・補強・比較・拡張のいずれかの役割を持つ
- できれば最近のものを優先
- ただし最近であることよりも、main 論文との関係の深さを優先
- 同じ内容の近縁論文だけで固めない
- 実験・理論・計算・材料系・応用先が少し分散するように選ぶ

検索で得た論文も、以下の役割カテゴリで整理すること。
- 背景
- 補強
- 比較
- 異手法
- 波及先
- 応用

### Step 5. 使う論文群の最終整理
最終的に記事で実質的に使う論文群を以下のように整理すること。
- main 論文: 1本
- references フォルダ内の関連論文: 2〜数本
- arXiv 検索で追加した論文: およそ 10~20 本前後

ただし、すべてを均等に扱わないこと。
扱いの重みは以下とすること。
1. main 論文
2. main と直接つながる sub 論文
3. main を深める検索論文
4. 広がりを示す補助的論文

## 図の取り扱い
--------------------------------

### 基本原則
図は「多めに使ってよい」が、必ずライセンス確認を行うこと。
適法性を最優先し、曖昧な場合は使わないこと。

### ライセンス確認手順
各論文について必ず以下を行うこと。
1. arXiv の abstract page でライセンス表記を確認する
2. 必要に応じて HTML 版、PDF 冒頭、著作権表記を確認する
3. 図の再利用が明確に許可されるか判定する
4. 判定結果を本文または図キャプションで attribution として示す

### 使用可否の原則
使用してよい:
- CC BY
- CC BY-SA
- CC0
- その他、今回の用途で再利用条件を満たせると明確に判断できるもの

慎重使用:
- CC BY-NC
- CC BY-NC-SA
- CC BY-ND
- CC BY-NC-ND

ただし、
- NC/ND 条件に少しでも不安があれば使わない
- ND の場合は改変禁止。トリミング、注釈追加、色変更、再構成は禁止

使用しない:
- arXiv default / non-exclusive license のみ
- ライセンス不明
- 再利用許可が曖昧

### ライセンス不適合時の代替
使いたい図が使えない場合は以下で対応すること。
1. その図は使わず、文章で丁寧に説明する
2. 著作権を侵害しないオリジナル概念図の案を作る
3. 必要なら「ここに概念図を自作するとよい」という形で markdown に挿入案を書く

### 図の選び方
優先順位:
1. main 論文の中心主張を最も端的に示す図
2. トピック全体の理解に役立つ概念図
3. 比較・相図・スペクトル・バンド・輸送特性など論点を説明しやすい図
4. 関連論文との違いが直感的に見える図
5. 他分野への接続が見える図

図は「論文ごと」に機械的に並べないこと。
章ごとの論点に最も対応する位置へ配置すること。

## 記事設計の基本方針
--------------------------------

### 最重要ルール
記事は「main 論文」「sub-1」「sub-2」「検索論文A」…という並列要約型にしないこと。
論文ごとの独立節を作らないこと。

その代わり、
- なぜこの main 論文の問いが重要か
- main 論文は何を新しく示したか
- 関連研究を踏まえると、その主張はどこまで強いか
- どこに比較対象があるか
- どの解釈にまだ不確かさがあるか
- 他材料系・他手法・応用へどう広がるか
という論点主導で章立てを設計すること。

### main 論文の位置づけ
main.pdf は記事全体の核であり、
- タイトル
- 導入
- 問いの設定
- 主結果の説明
- 論点整理
のすべてにおいて中心に置くこと。

検索論文は、
「main 論文の周辺にある論点を見通しよくするための材料」
として使うこと。

### 関連論文の使い方
関連論文は、各章の中で必要に応じて引用すること。
役割の例:
- 導入章で背景を与える
- 主結果の章で比較対象として用いる
- 解釈の章で異なる立場を並べる
- 展望の章で波及先を示す

### 3. main 論文は何を新しく示したのか

## 「基礎から理解する」の要件
--------------------------------
- 学部4年生が理解できるように書くこと
- まず直感的説明を書き、その後に正確な説明へ進むこと
- 必要に応じて数式を用いること
- 数式中の記号は必ず説明すること
- main 論文で前提になっている理論・概念を優先的に解説すること

--------------------------------
## 「重要キーワード10個」の要件
--------------------------------
- main 論文を理解するうえで重要なものを選ぶこと
- 単なる単語集ではなく、定義・意味・文脈を説明すること
- 可能なら main 論文のどの論点とつながるかを示すこと
- 必要に応じて簡単な数式を使ってよい

--------------------------------
## 文体上の注意
--------------------------------
- 日本語として自然で説明的に書くこと
- 単なる抄録の言い換えにしないこと
- 専門用語は初出時に定義すること
- 過剰な断定を避けること
- 実験・理論・計算の関係が見えるようにすること
- 記事全体を 1 本のストーリーにすること
- 主語は「論文」ではなく「問い」「論点」に寄せること
- ただし main 論文の中核結果だけは明確に中心に据えること

--------------------------------
## 最後に必ず行うこと
--------------------------------
記事を書き始める前に、短く内部整理してから進むこと。
以下を明確にしてから執筆すること。
1. main 論文の中心主張は何か
2. main 論文の議論を深めるのに必要な関連論文はどれか
3. 追加検索では何を補うべきか
4. 記事全体の中心メッセージを 1〜2 文で言うと何か

この内部整理を踏まえて、最終的な Markdown 記事を出力すること。
中間メモを長々と見せる必要はないが、記事の構成にはその整理を反映すること。
